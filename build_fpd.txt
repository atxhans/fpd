You are Claude Code. Build a production-ready full-stack B2B SaaS application for Fieldpiece called:

FIELDPIECE DIGITAL

Fieldpiece Digital is a multi-tenant HVAC intelligence and service platform that extends Fieldpiece’s position as the trusted tool brand for HVACR technicians into software, workflows, data capture, analytics, and support operations.

This is NOT a single-tenant app. It must be designed from day one as a true multi-tenant B2B platform where Fieldpiece operates the platform centrally and HVAC service companies are tenant customers.

The application should be branded as “Fieldpiece Digital” and the design language should be inspired by the visual identity of fieldpiece.com:
- industrial / professional / field-service oriented
- modern and high-trust
- bold, clean layouts
- color palette, visual tone, and component styling influenced by fieldpiece.com
- use a UI system that feels like a natural digital extension of the Fieldpiece brand
- create a theme layer with reusable brand tokens so the palette can be centrally managed

Do not copy the website directly, but clearly model the UI mood, hierarchy, and palette after the Fieldpiece brand.

PRIMARY GOAL

Create a Fieldpiece Digital platform with:

1. A mobile-friendly technician application

==================================================
THIS MUST BE A TRUE MULTI-TENANT B2B APP
==================================================

This is a real multi-tenant SaaS application.

Requirements:
- each HVAC service company is its own tenant/account
- users belong to a tenant
- tenant data must be isolated
- Fieldpiece operates the platform as the central provider
- Fieldpiece internal users need elevated admin/support capabilities
- future analytics may operate on anonymized/aggregated cross-tenant data, but raw tenant data must remain protected

Use strong tenant boundaries throughout:
- schema design
- queries
- APIs
- UI
- RBAC
- audit logs
- support tooling

Use Supabase Row Level Security where appropriate.
